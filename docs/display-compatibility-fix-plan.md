# MyQuickPasteSlint 跨 PC 显示兼容性修复计划

## 摘要
- 本轮只解决窗口跨 PC 显示不稳定，不混入主页视觉微调。
- 目标问题：
  - `Home` / `Settings` 首次打开或托盘打开时，窗口可能部分落到屏幕外，尤其右侧被裁。
  - `Settings` 在部分机器上出现黑边、透明壳暴露、内容像只显示一半。
- 当前根因：
  - 现有定位函数本质上是固定贴右停靠，不是基于目标点再做约束。
  - `show -> restore/bring-to-front -> timer 纠偏` 存在竞态，系统可能恢复到旧位置。
  - 目标显示器只按鼠标所在屏选取，没有按实际窗口所在屏做二次确认。
  - 顶层仍是 `no-frame` 自绘窗体，不同 Windows / DPI / GPU 环境下更脆弱。
- 本轮策略：兼容稳定性优先，统一修复 `Home` 与 `Settings` 的窗口显示链路。

## 关键改动
### 1. 重写窗口显示与定位链路
- `show_component_window()` 改为两阶段流程：
  - 阶段一：按设计尺寸与初始工作区计算候选尺寸和候选位置。
  - 阶段二：窗口显示并激活后，读取真实外框矩形，再按真实窗口所在显示器做最终尺寸和位置修正。
- 禁止继续使用“先定位，再 `SW_RESTORE`，再靠 timer 补救”的顺序。
- `bring_window_to_front()` 只负责激活和置前，不再承担会改变位置语义的恢复动作。
- 首次启动主页继续放在事件循环中执行，但最终位置以后验真实外框为准。

### 2. 将定位函数从“贴右停靠”改成“真正的约束”
- 拆分定位逻辑：
  - 先计算“靠右居中”的目标位置。
  - 再根据窗口矩形和工作区做 clamp。
- 默认锚点仍保持“靠右居中”的产品意图，但实现必须先算目标、再约束。
- 安全边距统一，主页和设置页共用一套规则。

### 3. 目标显示器改为“实际窗口优先”
- 预估阶段仍可用鼠标所在屏作为初始显示器。
- 显示后如果真实窗口落到另一块显示器，最终修正必须以该显示器工作区为准。
- 平台层补充：
  - 当前进程窗口句柄解析
  - 基于窗口句柄读取外框矩形
  - 基于窗口句柄读取所在显示器工作区
- 不再只靠 `FindWindowW(None, title)` 的全局命中结果做判断。

### 4. 收口顶层壳体兼容性风险
- `AppWindow` 与 `SettingsWindow` 保持统一壳体策略。
- 顶层窗口继续允许无边框，但不依赖透明外层去表现圆角或留白。
- 外层背景保持完整实底，内部留白全部放到内层容器。
- `Settings` 继续保持滚动主体结构，不回退到固定高度堆叠。

## 接口与实现约束
- Rust 侧新增或重构以下能力：
  - 当前进程窗口句柄解析
  - 基于窗口句柄读取外框矩形
  - 基于窗口句柄读取所在显示器工作区
  - 基于目标锚点计算位置，再执行统一 clamp
- UI 层保持当前结构，不新增业务功能，不改配置结构。
- 设计基准尺寸保持：
  - `Home`: `440x664`
  - `Settings`: `368x456`
- 最终显示尺寸规则保持：
  - `final_size = min(design_size, work_area - safe_margin * 2)`
- 本轮不引入窗口位置持久化。

## 测试与验收
### 代码级验证
- `cargo check`
- `cargo test`
- `cargo build --release`
- 为纯函数新增测试：
  - 工作区大于设计尺寸时保持设计尺寸
  - 工作区宽不足时横向缩窗
  - 工作区高不足时纵向缩窗
  - 目标位置在右侧越界时可正确 clamp 回可见区
  - 真实外框大于预估尺寸时仍能完整可见

### 手工场景
- 单屏：
  - `1920x1080 @ 100%`
  - `1920x1080 @ 125%`
  - `1920x1080 @ 150%`
  - `1366x768`
- 双屏：
  - 双屏不同缩放
  - 鼠标停在副屏，从托盘打开主页
  - 鼠标停在主屏，从托盘打开设置
- 特殊流程：
  - 应用首次启动主页必须完整可见
  - 托盘打开主页不得右侧出屏
  - 托盘打开设置不得出现黑边或透明壳暴露
  - 设置页状态条出现时允许滚动，不允许裁切

### 验收标准
- 不允许 `Home` 或 `Settings` 任一边落到屏幕外。
- 不允许首次启动“窗口已打开但用户看不到”。
- 不允许黑边、透明边、半刷出状态。
- 主页与设置页必须共用同一套窗口边界安全策略。

## 默认假设
- 本轮计划单独成文，不覆盖现有 `docs/ui-refactor-plan.md`。
- 优先修复共享窗口定位链路，不先做视觉层面的细修。
- 允许继续使用无边框窗口，但必须以稳定显示为最高约束。
- 若显示后真实外框与预估不一致，以真实外框为最终定位依据。
